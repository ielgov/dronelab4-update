Options for creating a new application in OpenShift

Within the OpenShift web console Developer perspective, there is the +Add item in the left navigation menu. This page is the entry point for adding an application or service to your OpenShift project.

The Add page offers (6) user flows for adding components from Git, deploying Container Images, adding an item from the Developer Catalog, importing your Dockerfile from a git repo, Importing YAML or adding a Database. You can create, build and deploy applications easily and in real-time using these user flows. From a developer's point of view, deploying an application to OpenShift is easy. The challenging part is finding the best way to do it because OpenShift is so adaptable and flexible.

You should consider the following when determining which way is the best when deploying an application: your development process, your developers' maturity regarding containers, your ops or technical experts' involvement, etc.

  • From Git
    This option involves importing code from your git repository to be built and deployed.
  • Container Image
    This option involves deploying an existing image from an image registry or image stream tag.
  • From Catalog
    This option involves browsing the catalog to discover, deploy and connect to services.
  • From Dockerfile (*Covered in this lab)
    This option involves importing your Dockerfile from your git repo to be built & deployed.
  • YAML
    This option involves creating resources from YAML file or JSON definitions.
  • Database
    This option involves browsing the catalog to discover database services to add to your application.
It is recommended that lab users take a look at https://access.redhat.com/documentation/en-us/openshift_container_platform/4.2/html/applications/application-life-cycle-management#odc-creating-applications-using-developer-perspective and https://docs.openshift.com/container-platform/4.3/applications/application_life_cycle_management/odc-creating-applications-using-developer-perspective.html to get an in-depth summary of each of these options not covered in the lab. Due to time limitations and the nature of labs, the “From Dockerfile” option was best suited for a lab user to quickly deploy an application to the hybrid multi-cloud while still demonstrating OpenShift’s capabilities.
